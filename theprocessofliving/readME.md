# one fortiethly

### ASCII Fortune Teller: The Process of Life _(in progress)_

---

## what is this????

A browser-based interactive divination system built with vanilla JavaScript, HTML, CSS, and p5.js. The user consults an ASCII Magic 8-ball. Most of the time it gives a standard response. Occasionally it opens a using a full 52-card cartomancy deck. The project is organized around three temporal states **PAST**, **PRESENT**, **FUTURE** of which only **FUTURE** is built in this iteration.

---

## billy ball again?

The `8ball.jpg` image is resized to match the character grid. Each pixel's brightness maps to a character from the string `Ñ@#W$9876543210?!abc;:+=-,._ ` dark pixels become dense characters, bright pixels become sparse ones. Hovering near cells permanently reveals the image character underneath. Clicking inside the ball picks a response from the weighted array defined in `index.html` as a JSON block.

**FUTURE** appears 8 times in a pool of 56 responses, giving roughly a 1-in-7 chance per click. Normal responses appear twice each.

When **FUTURE** is picked the answer scrolls through the center band of the ball in red for 1.7 seconds, then `StateManager` switches the screen

## card rituals

Managed by the `OneForth` class in `oneforth.js`. The current phase is tracked using a `custom-bool` attribute on `#future-stage`. Each click reads this attribute to decide what to do.
Meaning is generated by `CardReading` in `cardreadings.js`. Each card contributes based on:

- **Suit** - Base register (EXAMPLE Hearts: love; Diamonds: movement; Clubs: effort; Spades: difficulty)
- **Rank** - intensity and theme (Ace: beginning; King: mastery, etc.)
- **Position** - some basic "what has shaped you / what surrounds you now / what is moving toward you" logic
  The three outputs are: a short summary line, an oblique narrative sentence, and a longer per-card interpretation.

## what is still in progress

- **PAST state** — MIDI keyboard playthrough connected to Max/MSP, user builds custom synth patches from a constrained key set
- **PRESENT state** — real-time sensor/camera interaction, particle system, face filter
- **Video content** — shuffle and split sequences will use recorded footage of hands handling a physical deck, replacing the current `setTimeout` placeholders
- **Full cartomancy text** — rank and suit meanings are placeholder strings, to be replaced with sourced readings from the cartomancy research corpus
- **Multiplayer logic** — PAST output logged and passed to the next user's PRESENT session
- **Return button** — wired in `main.js`, functional
